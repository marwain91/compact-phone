# v0.1 Internal Alpha — Acceptance Procedure

**Prerequisites:**
- macOS 12+ Apple Silicon machine
- Xcode CLI tools, Homebrew, Docker Desktop running
- Project checked out, `VCPKG_ROOT` pointing at a vcpkg checkout

## Headless test loop (Linux dev container)

These steps validate the SIP signaling and audio plumbing without any UI or real audio device. They run inside the dev container.

```bash
docker compose -f tools/dev/docker-compose.yml up -d
docker compose -f tools/dev/docker-compose.yml exec -T dev cmake --preset linux
docker compose -f tools/dev/docker-compose.yml exec -T dev cmake --build --preset linux
docker compose -f tools/dev/docker-compose.yml exec -T dev \
    bash -c "cd /workspace/build/linux && ctest --output-on-failure"
```

Expected: 2 tests pass — `unit` (SipEngine lifecycle) and `integration` (UDP register + echo-call round-trip against the bundled Asterisk).

## Native macOS audio loop (the actual acceptance test)

This is the real v0.1 acceptance — it exercises the audio path on real hardware.

1. Build natively on macOS:
   ```bash
   brew install cmake ninja pkg-config
   curl -sL https://github.com/pjsip/pjproject/archive/refs/tags/2.17.tar.gz | tar -xz
   cd pjproject-2.17
   ./aconfigure --prefix=$HOME/pjproject --disable-shared --enable-static \
       --with-ssl=yes --disable-video --disable-libwebrtc --disable-libyuv \
       --disable-opencore-amr --disable-bcg729 --disable-sdl --disable-ffmpeg \
       --disable-v4l2 --disable-openh264
   make dep && make -j$(sysctl -n hw.ncpu) && make install
   cd ..
   export PKG_CONFIG_PATH=$HOME/pjproject/lib/pkgconfig
   cmake --preset macos
   cmake --build --preset macos
   ```

2. Start the Asterisk fixture (uses the same compose file):
   ```bash
   docker compose -f tools/dev/docker-compose.yml up -d asterisk
   ```

3. Launch the app:
   ```bash
   ./build/macos/src/compactphone.app/Contents/MacOS/compactphone
   ```

4. macOS shows the microphone-permission prompt (sourced from `NSMicrophoneUsageDescription` in `Info.plist.in`). Click **Allow**.

5. In the **Account** group:
   - Username: `1001`
   - Password: `compactphone1001`
   - Server: `127.0.0.1:5060`
   - Click **Connect**. Within ~5 seconds, **Status** becomes `registered`.

6. In the **Dial** group, leave the default `sip:600@127.0.0.1:5060` and click **Call**:
   - **Call** moves through `calling` → `active`.
   - You hear Asterisk's demo-echotest prompt ("This is an audio echo test...").
   - Speaking into the microphone produces audible echo within ~1 second.

7. Click **Hang up**. **Call** returns to `idle`.

8. Quit the app (Cmd-Q). Stop Asterisk:
   ```bash
   docker compose -f tools/dev/docker-compose.yml down
   ```

## Pass criteria

All of the following:

- [ ] All headless tests pass (Linux container, ctest output 100%).
- [ ] Registration succeeds without manual retry on the macOS host.
- [ ] Outbound call to extension 600 connects and audio flows both ways.
- [ ] Hang-up completes within 2 seconds.
- [ ] No crash or unhandled exception in `spdlog` output.
- [ ] App quits cleanly on Cmd-Q (no zombie PJSIP threads — check Activity Monitor).

## Out of scope for v0.1

Each of these is intentionally deferred to a subsequent plan:

- TLS / SRTP
- Hold, mute, transfer, DTMF
- Contacts, history, settings UI
- macOS signing + notarization, Windows MSI
- Auto-update
- Multi-account
- Inbound call handling beyond signaling
